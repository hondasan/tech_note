SQL Server の下位互換性：できること・できないことまとめ

SQL Serverの運用において、バージョンのアップグレードやダウングレードに伴う「データベースの互換性」は非常に重要なテーマです。特に、「新しいバージョンで作成したバックアップを古いバージョンに戻せるか」といった下位互換性の制限について整理します。

1. 理解の前提：「エンジンバージョン」と「互換性レベル」の違い

SQL Serverには、混同しやすい2つの概念があります。

  • SQL Server エンジンのバージョン: インスタンス自体の製品バージョン（例：SQL Server 2017, 2019, 2022など）。物理的なデータベースファイルの構造や内部メタデータを決定します。
  • データベース互換性レベル (Compatibility Level): データベースごとの設定値（例：140, 150, 160など）。T-SQLの構文解釈やクエリ最適化の動作について、特定の古いバージョンの動作をエミュレートします。

重要： 互換性レベルを古いバージョン（例：2016相当の130）に設定していても、そのデータベースを実行しているエンジン自体が新しいバージョン（例：2022）であれば、物理的な下位互換性があるわけではありません。

2. できること・できないこと（比較一覧）

バージョン互換性に関する主な操作のサポート状況です。

操作 サポート可否 詳細
古い → 新しいバージョンへのリストア / アタッチ 〇 可能（前方互換） 古いSQL Serverのバックアップ（.bak）を新しいSQL Serverにリストアできます。
新しい → 古いバージョンへのリストア / アタッチ ✕ 不可（下位互換なし） 新しいSQL Serverのバックアップを古いSQL Serverにリストアすることは物理的にできません。アタッチも同様です。
リストア後の互換性レベルの維持 〇 原則可能 移行後も互換性レベルは維持されます。ただし、移行先エンジンがサポートする最小レベル未満の場合は、自動的に最小サポートレベルに引き上げられます。
新バージョンにアタッチしたDBの旧バージョンへの差し戻し ✕ 不可 一度新しいエンジンにアタッチされたデータベースは、内部メタデータが更新されるため、互換性レベルが古い状態であっても、古いエンジンに戻すことはできません。

3. なぜ「新しい → 古い」へのリストアはできないのか？

SQL Serverはデータベースを起動、あるいはアタッチする際、データベースの内部メタデータやストレージ構造を最新バージョンのフォーマットに自動的にアップグレードします。このアップグレード処理は一方向であり、古いバージョンには存在しない新機能のメタデータやストレージ構造（インデックスやシステムテーブルの変更）が追加されるため、古いエンジンではこれを解釈・認識できません。

4. 新しいバージョンから古いバージョンへ移行する回避策

ネイティブのバックアップとリストアが使用できないため、どうしてもダウングレードや旧環境へのデータ移行が必要な場合は、以下の方法を検討します。

方法 A: スクリプトの生成 (Generate Scripts)

SQL Server Management Studio (SSMS) を使用して、スキーマとデータをSQLスクリプトファイルとして出力し、移行先で実行します。

  1. 対象のデータベースを右クリック ＞ [タスク] ＞ [スクリプトの生成] を選択。
  2. 詳細設定のオプションで「スクリプトを作成するデータベースエンジンの版」に [移行先の古いSQL Serverのバージョン] を指定。
  3. 「スクリプトを作成するデータの種類」で [スキーマとデータ] または [データのみ] を指定。
  4. 生成されたSQLファイルを移行先の古いSQL Serverインスタンスで実行する。

方法 B: BACPAC ファイルのエクスポート / インポート

スキーマ定義とデータをまとめてパッケージングする Data-tier Application (DACPAC / BACPAC) を利用します。

  • SSMS等で [データ層アプリケーションのエクスポート] を行い、.bacpac ファイルを作成します。
  • 移行先の古いSQL Serverで [データ層アプリケーションのインポート] を行い、データベースを再構築します。

5. まとめ

SQL Serverでは、古いバージョンから新しいバージョンへの移行（前方互換）は柔軟に行える一方で、新しいバージョンから古いバージョンへのリストア（下位互換）は一切サポートされていません。移行計画や検証の際は、バックアップファイルからの切り戻しができない点に十分注意する必要があります。
