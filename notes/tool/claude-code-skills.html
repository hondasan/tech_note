Claude Codeの「Skills」とは？仕組み・作り方・ユースケースからCodex/Antigravityとの違いまで解説

自律型のAIコーディングエージェントを使いこなす上で、繰り返し行う複雑な作業をどのように自動化するかは重要なテーマです。Anthropicが提供するCLIツール「Claude Code」には、こうした要件を満たすための「Skills（カスタムスキル）」という仕組みが備わっています。この記事では、Skillsの概念や具体的な作成方法、実践的なユースケースに加えて、他の主要なコーディングエージェント（Codex CLIやAntigravity）における対応状況を解説します。

1. Skillsとは？なぜ必要なのか

Claude Codeにおける「Skills」とは、AIエージェントにプロジェクト独自の開発手順や繰り返しのワークフローを指示するための、パッケージ化された指示セット（フォルダおよびMarkdownファイル）です。特定の処理をさせたいときに呼び出して実行させることができます。

プロジェクト全体の基本ルールを規定する CLAUDE.md と比較すると、Skillsには以下のような明確な役割とメリットがあります。

コンテキスト（トークン）の節約

CLAUDE.md に記載されたルールは、AIとチャットするたびに毎回すべてのコンテキストとして読み込まれます。そのため、大量の指示を書き込むとコンテキストが肥大化し、APIのトークン消費量と実行コストが増加します。一方、Skillsはそのスキルが呼び出されたときだけ一時的に指示をロードするため、必要なときにだけコンテキストを消費して効率的に動作します。

モジュール化による責務の整理

「コードのレビュー」「テストデータの生成」「ドキュメントの作成」といった個別のタスクごとに指示をフォルダ単位で分離して管理できます。これにより、指示のバグや競合を防ぎ、メンテナンスしやすくなります。

2. Claude Codeでのカスタムスキルの作り方

カスタムスキルを作成する方法には、対話的なアプローチと手動でのファイル作成の2通りがあります。

方法①：組み込みの `/skill-creator` を利用する

最も簡単なのは、CLI内で対話型作成ツールを利用する方法です。以下のコマンドを実行します。

/skill-creator

実行すると、AIが「どのような自動化ワークフローを作りたいか」をインタビュー形式で質問してきます。質問に答えていくことで、AIが必要なファイル群と指示書をバックグラウンドで自動的に組み立ててくれます。

方法②：手動でフォルダと `SKILL.md` を作成する

自身で詳細な指示を定義したい場合は、プロジェクトのルートディレクトリに手動で配置します。

  • ディレクトリ配置：プロジェクト内に .claude/skills/[任意のスキル名]/ というフォルダを作成します。
  • ファイルの作成：そのフォルダ内に SKILL.md というファイルを配置します。

SKILL.md の中身は、冒頭にYAML形式のメタデータ（フロントマター）を記述し、その後に具体的な実行指示を書き込みます。

---
name: "code-reviewer"
description: "PRのコードスタイルとセキュリティリスクを検証する"
---

# 指示内容
コミットされた差分を読み込み、以下の項目を検証してください。
- セキュリティ脆弱性の有無
- 関数の命名規則がプロジェクトの規約に沿っているか
- パフォーマンスのボトルネックになり得る箇所の指摘

作成したスキルは、ローカルから追加することができます。たとえば、以下のコマンドを実行することで、任意のディレクトリにあるスキルをインストールできます。

/plugin add /path/to/skill-directory

3. 実践的なユースケース

Skillsは以下のような定型的でありながら複雑な処理を自動化するのに適しています。

  • 差分ベースのチェンジログ生成：Gitの差分（diff）を解析し、特定のフォーマットで CHANGELOG.md を自動更新する。
  • テストデータの自動構築：スキーマ定義ファイルをインプットとして、テスト用のダミーデータ（JSONなど）を生成する。
  • 依存ライブラリのアップデート確認：パッケージ管理ファイル（package.jsonなど）をスキャンし、推奨される更新内容をまとめる。

4. Codex CLIやAntigravityにおけるスキルの扱い

同様の自律型コーディングエージェントにおいても、名前やアプローチは異なりますが、同様の自動化機能が提供されています。

OpenAI Codex CLIでの対応

以前のCodex CLIでは、~/.codex/prompts/ に配置したテキストファイルを読み込む「Custom Prompts」という機能が使われていました。現在は、同様のタスク指向の自動化機能として「Skills」システムに順次移行しています。これにより、繰り返しのタスク定義が容易になっています。また、非対話型で処理を回したい場合は、codex exec "タスクの記述" をシェルスクリプトやCI/CDに組み込む手法が推奨されています。

Antigravity（本エージェント）での対応

本コーディングアシスタント「Antigravity」では、ファイルを閲覧・読み込みするツール（view_file など）に IsSkillFile という専用のパラメータが用意されています。これは、実行手順が記載されたMarkdownファイル等を一時的に「スキル指示書」としてロードし、その指示内容に準拠してタスクを実行するための仕組みです。これにより、特定のプロジェクトの複雑な更新ルールをオンデマンドで読み込み、忠実に再現することができます。

5. まとめ

Claude Codeの「Skills」は、ただ指示を増やすだけでなく、「使いたいときに、使いたい分だけ」AIの行動をコントロールできる強力な仕組みです。チーム開発におけるコードスタイルの統一や、泥臭い定型タスクの自動化に向けて、まずはシンプルなカスタムスキルの作成から試してみることをおすすめします。
